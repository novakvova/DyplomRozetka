# Командний проєкт Rozetka

Мобільний застосунок `Rozetka`, зібраний на `Expo + React Native + TypeScript`.

Проєкт покриває основні користувацькі сценарії інтернет-магазину:
- вхід і реєстрація;
- відновлення та зміна пароля;
- профіль користувача;
- кошик;
- оформлення замовлення;
- доставка через Нову пошту;
- каталог товарів;
- пошук і фільтрація;
- картка товару;
- історія замовлень;
- вхід через Google;
- адмін-панель для керування товарами, категоріями і користувачами.

## Реалізовані модулі

1. Вхід на сайт
2. Реєстрація користувача з валідацією `email / password / confirm password`
3. Відновлення паролю
4. Зміна паролю користувача
5. Профіль користувача
6. Розробка кошику користувача
7. Додавання товарів у кошик
8. Зміна кількості товарів у кошику
9. Видалення товарів із кошика
10. Оформлення замовлень користувачем
11. Додавання інформації із Нової пошти для відправки замовлень
12. Розробка головної сторінки сайту. Відображення каталогу товарів
13. Пошук і фільтрація товарів
14. Картка перегляду товару
15. Система автоматичного заповнення локальної БД товарами та категоріями
16. Вхід через Google
17. Перегляд історії замовлення користувача
18. Адмін-панель для керування товарами, категоріями і користувачами

```bash
npm install

```

Альтернативні команди:

```bash
npm run ios
npm run android
npm run web
npm run typecheck
```

## Google Login

Для Google-входу потрібно створити локальний файл `.env` на основі шаблону:

```bash
cp .env.example .env
```

У `.env` потрібно вказати реальні значення:

```env
EXPO_PUBLIC_GOOGLE_WEB_CLIENT_ID=...
EXPO_PUBLIC_GOOGLE_IOS_CLIENT_ID=...
EXPO_PUBLIC_GOOGLE_ANDROID_CLIENT_ID=...
```

Важливо:
- `.env` не зберігається в репозиторії навмисно;
- у репозиторії є тільки `.env.example`;
- для native-перевірки Google-входу потрібен `development build` або `standalone app`;
- у `Expo Go` native Google-вхід не є повноцінним сценарієм перевірки.

## Доступ до адмін-панелі

Адмін-панель видно після входу під email:

```text
radon.bogdan09@gmail.com
```

Саме для цього email у проєкті за замовчуванням призначається роль `admin`.

Після входу:
1. відкрийте головну сторінку;
2. прокрутіть до блоку кнопок;
3. натисніть `Адмін-панель`.

У панелі доступно:
- додавання, редагування і видалення товарів;
- додавання, редагування і видалення категорій;
- перегляд користувачів;
- зміна ролей `user/admin`;
- видалення користувачів разом із локальними даними;
- скидання каталогу до seed-стану.

## Як перевірити основні сценарії

### Користувацький flow

1. Зареєструвати нового користувача
2. Увійти в акаунт
3. Заповнити профіль
4. Додати товари в кошик
5. Змінити кількість або видалити позицію
6. Перейти в checkout
7. Обрати місто і точку Нової пошти
8. Оформити замовлення
9. Відкрити `Мої замовлення`
10. Перевірити історію і повторне замовлення

### Каталог

1. Відкрити головну сторінку
2. Перемикати категорії
3. Використати пошук
4. Застосувати фільтр за брендом і ціною
5. Відкрити `Переглянути` для будь-якого товару

### Адмінка

1. Увійти під admin email
2. Відкрити `Адмін-панель`
3. Створити або змінити категорію
4. Створити або змінити товар
5. Перевірити, що зміни видно в каталозі
6. Переглянути користувачів і змінити роль іншому акаунту

## Локальне збереження

Проєкт працює через `AsyncStorage` і локально зберігає:
- сесію користувача;
- зареєстрованих користувачів;
- профілі;
- кошики;
- замовлення;
- локальну БД каталогу.

Це дозволяє демонструвати повний сценарій роботи без окремого backend.
